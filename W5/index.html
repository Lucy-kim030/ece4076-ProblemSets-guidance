Projection Matrix Decomposition

Recover K and [R|t] from P — one row at a time, with full reasoning for each step.

First — What Does P = K[R|t] Mean?

When we write P = K · [R | t], we are saying that the 3×4 matrix P is the product of two matrices. Let's see what that product looks like symbolically:

P = K · [R|t]

= [ f_u s u₀ ] × [ r₁₁ r₁₂ r₁₃ tₓ ] [ 0 f_v v₀ ] [ r₂₁ r₂₂ r₂₃ t_y ] [ 0 0 1 ] [ r₃₁ r₃₂ r₃₃ t_z ]
💡 Key insight before we start

The third row of K is [0, 0, 1]. So the third row of P = third row of K times [R|t] = 1×(third row of R, then t_z). This means the first 3 elements of P's 3rd row give us the 3rd row of R directly!

Step 1 — Read Off r₃ and Solve for v₀
1a

Why can we read r₃ directly from P's 3rd row?

📐 Why this works

Multiply K's 3rd row [0, 0, 1] by each column of [R|t]:
[0, 0, 1] · [r₁₁,r₂₁,r₃₁] = r₃₁
[0, 0, 1] · [r₁₂,r₂₂,r₃₂] = r₃₂
[0, 0, 1] · [r₁₃,r₂₃,r₃₃] = r₃₃
So the 3rd row of P = [r₃₁, r₃₂, r₃₃, t_z] — pure R's 3rd row! No K involved at all.

P's 3rd row = [0, 0, 1, 3] ↑ ↑ ↑ ↑ r₃₁ r₃₂ r₃₃ (will use later for t_z) ∴ [r₃₁, r₃₂, r₃₃] = [0, 0, 1]

This also tells us t_z = 3 (the 4th element of P's 3rd row). We get it for free!

1b

Solve for v₀ (the vertical principal point)

📐 Why this formula works

P's 2nd row = f_v·[r₂₁,r₂₂,r₂₃] + v₀·[r₃₁,r₃₂,r₃₃]
Dot-product both sides with r₃ = [0,0,1]:
P's 2nd row · r₃ᵀ = f_v·(r₂·r₃ᵀ) + v₀·(r₃·r₃ᵀ)
Since R is orthonormal: r₂·r₃ = 0, r₃·r₃ = 1. So the left side = v₀!

v₀ = P's 2nd row · [r₃₁, r₃₂, r₃₃]ᵀ = [280, 280, 240] · [0, 0, 1]ᵀ ↑only this term survives because r₃ = [0,0,1] = 280×0 + 280×0 + 240×1 v₀ = 240

v₀ = 240 means the optical axis hits the image at row 240 — the vertical center of the image (typical for a 480-pixel-tall image).

Step 2 — Solve for f_v and r₂
2a

Isolate f_v · r₂

📐 Why we subtract v₀·r₃

P's 2nd row = f_v·r₂ + v₀·r₃ (from the matrix multiplication). We already know v₀ and r₃, so rearranging gives us f_v·r₂. It's just algebra — move the known term to the other side.

f_v · r₂ = P's 2nd row − v₀ · r₃ = [280, 280, 240] − 240 × [0, 0, 1] = [280, 280, 240] − [0, 0, 240] = [280, 280, 0]
2b

Find f_v by taking the norm

📐 Why the norm gives f_v

We have f_v·r₂ = [280, 280, 0]. We know r₂ must be a unit vector (because R is a rotation matrix — all its rows have length 1). So:
‖f_v · r₂‖ = f_v · ‖r₂‖ = f_v · 1 = f_v
Just take the norm of [280, 280, 0]!

f_v = ‖[280, 280, 0]‖ = √(280² + 280² + 0²) = √(78400 + 78400) = √156800 = 280 × √2 f_v ≈ 396
2c

Recover unit vector r₂

r₂ = (f_v · r₂) / f_v = [280, 280, 0] / 396 = [280/396, 280/396, 0/396] = [0.7071, 0.7071, 0] ← this is sin45°, cos45°!
💡 Geometric meaning

r₂ = [0.707, 0.707, 0] means the camera's Y-axis points diagonally between world X and world Y at 45°. The camera is rotated 45° about the Z-axis.

Step 3 — Solve for u₀ (skew s) and r₁
3a

u₀ — same idea as v₀ but using P's 1st row and r₃

u₀ = P's 1st row · [r₃₁, r₃₂, r₃₃]ᵀ = [280, -280, 320] · [0, 0, 1]ᵀ = 280×0 + (-280)×0 + 320×1 u₀ = 320

u₀ = 320 → optical axis hits column 320 (horizontal center of a 640-wide image). ✓

3b

Skew s — dot P's 1st row with r₂

📐 Why

P's 1st row = f_u·r₁ + s·r₂ + u₀·r₃. Dot with r₂:
= f_u·(r₁·r₂) + s·(r₂·r₂) + u₀·(r₃·r₂) = s
Because r₁⊥r₂, r₂·r₂=1, r₃⊥r₂ (orthonormality of R).

s = P's 1st row · r₂ᵀ = [280, -280, 320] · [0.7071, 0.7071, 0]ᵀ = 280 × 0.7071 + (-280) × 0.7071 ← these cancel! + 320 × 0 = 197.99 − 197.99 + 0 s = 0

s = 0 means pixels are perfectly rectangular — no shear between axes. Normal for modern cameras.

f_u · r₁ = P's 1st row − u₀·r₃ − s·r₂ = [280, -280, 320] − 320×[0, 0, 1] − 0×[0.707, 0.707, 0] = [280, -280, 320] − [0, 0, 320] = [280, -280, 0] f_u = ‖[280, -280, 0]‖ = √(280²+280²) = 396 r₁ = [280, -280, 0] / 396 = [0.7071, -0.7071, 0]

📐 What is the 4th column of P?

The 4th column of P comes from K·t (where t = [tₓ, t_y, t_z]ᵀ is the camera translation). Expanding K·t:

K · t = [ f_u·tₓ + s·t_y + u₀·t_z ] ← = p₁₄ = 1360 [ f_v·t_y + v₀·t_z ] ← = p₂₄ = 1120 [ t_z ] ← = p₃₄ = 3

Q2

Radial Distortion

Why we divide by z, what r² means, and how a single number a reshapes an entire image.

The 4-Step Pipeline — Full Reasoning
🎥 The situation

We have a 3D point (x, y, z) = (1, 2, 4) in camera coordinates (already rotated/translated). We want its pixel location, but the lens distorts light before it hits the sensor.

3D point (x,y,z) → normalize → distort → project via K → pixel (u,v)
Step 1 — Normalize by z (Pinhole Projection)
📐 Why divide by z?

The pinhole camera model says: a ray from (x,y,z) through the focal point hits the image plane at depth f. By similar triangles: the image coordinate is proportional to x/z (and y/z). Dividing by z converts 3D camera coordinates to 2D angles. Think of it as projecting onto a "normalized image plane" at distance 1 from the camera.

Given: (x, y, z) = (1, 2, 4) u = x / z = 1 / 4 = 0.25 ← normalized horizontal coordinate v = y / z = 2 / 4 = 0.50 ← normalized vertical coordinate These are the coordinates on the normalized image plane (z=1).

Units here are NOT pixels yet — u and v are dimensionless ratios. They represent angles roughly: tan(θ_x) ≈ 0.25 means about 14° from the optical axis.

Step 2 — Compute r² (Radial Distance Squared)
📐 Why do we need r²?

Radial distortion is symmetric around the optical axis. The amount of distortion depends only on how far the ray is from the center, not on the direction. That distance on the normalized plane is r = √(u²+v²), and the distortion model uses r².

r² = u² + v² = 0.25² + 0.50² = 0.0625 + 0.2500 = 0.3125 (r = √0.3125 ≈ 0.559 — the point is 0.559 "units" from the image centre)

We use r² (not r) in the formula because it avoids a square root and the polynomial r(1+ar²) is easier to work with mathematically.

Step 3 — Apply Distortion: r′ = r(1 + ar²)
📐 What does this formula DO?

The formula r′ = r(1 + ar²) is a radial stretching/shrinking of the image. Each point (u,v) is pulled toward or away from the center by the factor (1 + ar²):
• If a < 0: factor < 1 → points move inward → barrel (fisheye look)
• If a > 0: factor > 1 → points move outward → pincushion
• The center (r=0) is never distorted because 1 + a×0 = 1.

-0.10
a = −0.1, r² = 0.3125 scale = 1 + a × r² = 1 + (−0.1) × 0.3125 = 1 − 0.03125 = 0.96875 ← less than 1 → barrel (inward shift) u′ = u × scale = 0.25 × 0.96875 = 0.242 (was 0.25, moved 0.008 toward center) v′ = v × scale = 0.50 × 0.96875 = 0.484 (was 0.50, moved 0.016 toward center)

Notice v′ moved more than u′ because v is larger (farther from center vertically), so distortion affects it more.

Step 4 — Project to Pixels via K
📐 What does K do here?

K converts the normalized distorted coordinates (u′, v′) to pixel coordinates. It scales by the focal lengths (f_u, f_v) and shifts by the principal point (u₀, v₀). Think of f_u as "how many pixels per unit of normalized coordinate".

K = [ 600 0 320 ] [ 0 600 240 ] [ 0 0 1 ] pixel_x = f_u × u′ + u₀ = 600 × 0.242 + 320 = 145.1 + 320 = 465.3 pixel_y = f_v × v′ + v₀ = 600 × 0.484 + 240 = 290.6 + 240 = 530.6

Vanishing Points & Lines

Homogeneous coordinates, how infinity becomes finite, and the horizon line equation.

Background — Homogeneous Coordinates (crucial!)
💡 What are homogeneous coordinates?

A 2D point (x, y) is represented as [x, y, 1]ᵀ in homogeneous form. After matrix multiplication you might get [a, b, c]ᵀ with c ≠ 1. The actual 2D point is then (a/c, b/c). This lets us represent points at infinity: when c → 0, the point moves to infinity!

Homogeneous [a, b, c]ᵀActual 2D pointMeaning
[0, 0, 10](0/10, 0/10) = (0, 0)Origin
[0, 20, 12](0/12, 20/12) = (0, 1.67)Finite point
[10, 10, 1](10, 10)Vanishing point VP₁
[1, 0, 0]x=1/0 = ∞Point at infinity in x!

Q3(3) — Points at Infinity → Vanishing Points
📐 How do we represent a direction (point at infinity)?

A point infinitely far in the x-direction is (∞, 0). In homogeneous form: as x→∞, the point is [1, 0, 0]ᵀ (the limit of [x, 0, 1/x] as x→∞). Notice w=0, so it cannot be normalized to a finite 2D point — unless we apply H first, which can make w≠0!

Q3(4) — Horizon Line Through VP₁ and VP₂
📐 The determinant method for a line through two points

A line through points (x₁,y₁) and (x₂,y₂) satisfies: any point (x,y) on the line makes the determinant of a 3×3 matrix zero. This is just saying the three points are collinear.

VP₁ = (10, 10), VP₂ = (−10, 10) A point (x, y) is on the horizon line iff it is collinear with VP₁ and VP₂: det | x y 1 | = 0 | 10 10 1 | | -10 10 1 | Expand along the first row: = x · det|10 1| − y · det|10 1| + 1 · det|10 10| |10 1| |-10 1| |-10 10| = x · (10×1 − 1×10) − y · (10×1 − 1×(−10)) + 1·(10×10 − 10×(−10)) = x · (10 − 10) − y · (10 + 10) + 1 · (100 + 100) = x · 0 − y · 20 + 200 = 0 Divide by 20: −y + 10 = 0 → y = 10 In projective form: [0, −1, 10]ᵀ meaning: 0·x − 1·y + 10 = 0
💡 Why y = 10?

Both vanishing points have y-coordinate = 10. So the horizon is simply the horizontal line at height y = 10 in the image. This makes sense: all points at infinity map to the same horizontal line.

Q3(5) — Verify Horizon via H⁻ᵀ
📐 Why H⁻ᵀ for lines?

Points transform as x′ = H·x. Lines (as dual vectors) transform differently: l′ = H⁻ᵀ·l. The line at infinity in the world is [0,0,1]ᵀ (it contains all directions w=0). Applying H⁻ᵀ maps it to the horizon in the image.

H⁻ᵀ = [ 0.05 0.05 0 ] [-0.05 0.05 0 ] [ 0 -0.01 0.1 ] H⁻ᵀ · [0, 0, 1]ᵀ: Row 1: 0.05×0 + 0.05×0 + 0 ×1 = 0 Row 2:-0.05×0 + 0.05×0 + 0 ×1 = 0 Row 3: 0 ×0 +(-0.01)×0 + 0.1×1 = 0.1 ← wait this is [0, 0, 0.1]? No — wait, let me recheck the rows... H⁻ᵀ = (H⁻¹)ᵀ: = [ 0.05 -0.05 0 ]ᵀ ← column 1 of H⁻¹ [ 0.05 0.05 -0.01 ]ᵀ ← column 2 of H⁻¹ [ 0 0 0.1 ]ᵀ ← column 3 of H⁻¹ H⁻ᵀ · [0, 0, 1]ᵀ = 3rd column of H⁻ᵀ = 3rd row of H⁻¹ = [0, -0.01, 0.1]ᵀ Scale by 10: [0, -1, 10]ᵀ ← matches our horizon equation! ✓
Q4

Cross Ratio

What the formula means, why it stays constant under projection, and the Law of Sines proof.

What Is the Cross Ratio?
💡 Intuition

Most measurements change under perspective (lengths, angles, simple ratios). The cross ratio is a special ratio of ratios that stays the same. It's like a "projective fingerprint" of four points.

CR(R₁, R₂, B, T) = |R₁T| · |R₂B| / (|TB| · |R₁R₂|)

Where |AB| means the distance from A to B. Let's build up intuition with a concrete number example first, then prove it.

30°

Move the slider to change the projection angle. Watch how individual distances change, but the cross ratio stays constant!

The Law of Sines Proof — Step by Step
📐 Setup

All 4 world points and their image projections lie on lines through the camera centre C. This set of lines is called a pencil of lines. Label the angles between consecutive lines as α, β, γ (see diagram below).

1

Express world distances using Law of Sines

📐 Law of Sines reminder

In any triangle, a/sin(A) = b/sin(B), where a is the side opposite angle A.

Consider triangle C-R₁-T (vertices at camera C, world R₁, world T): • The angle at C between CR₁ and CT is (α+β+γ) → wait, let me use: • α = angle between ray CR₁ and ray CB • β = angle between ray CB and ray CT • γ = angle between ray CT and ray CR₂ Using Law of Sines in triangle C-R₁-T: |R₁T| / sin(angle at C) = |CR₁| / sin(angle at T) After careful application to all four triangles: |R₁T| = k · sin(α+β+γ−something)... Simplification (solution shortcut): |R₁T| / |R₁B| = sin(α+β) / sin(α) ... (ratio in same triangle fan) |R₂B| / |R₂T| = sin(α+β) / sin(α+β+γ) So: |R₁T|·|R₂B| / (|TB|·|R₁R₂|) = [sin(α+γ)/sin(α)] · [sin(α+β)/sin(α+β+γ)]
2

Same expression for image points

The image points R₁′, R₂′, B′, T′ lie on a DIFFERENT line (the image plane), but their rays through C have the SAME angles α, β, γ! |R₁′T′|·|R₂′B′| / (|T′B′|·|R₁′R₂′|) = [sin(α+γ)/sin(α)] · [sin(α+β)/sin(α+β+γ)] SAME FORMULA. Therefore: World CR = Image CR ∴ Cross Ratio is INVARIANT ✓
💡 The deep reason

The cross ratio depends only on the angles at C — it doesn't matter which plane you intersect the pencil with. All planes give the same cross ratio.

3

Practical use: measuring unknown height h

Given: R₁ = ground level (0), R₂ = top of reference pole (H_ref = known) B = base of unknown object, T = unknown top From the image, measure: |R₁′T′|, |R₂′B′|, |T′B′|, |R₁′R₂′| Set up equation (world CR = image CR): |R₁T|·|R₂B| / (|TB|·|R₁R₂|) = (measured image CR) World distances: |R₁T| = h, |R₂B| = H_ref-b, |TB| = h-b, |R₁R₂| = H_ref (where b = height of base B above ground) Solve for h → unknown height from a single photo!

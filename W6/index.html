Why Two Cameras? — The Depth Ambiguity Problem
A single camera loses depth information

The pinhole camera model (lecture notes §1) projects 3D point x = [x₁, x₂, x₃]¹ to a 2D image using normalised coordinates (focal length = 1):

p₁ = x₁ / x₃  p₂ = x₂ / x₃
The problem: Depth x₃ appears in the denominator and cancels. Any point λ·[p₁, p₂, 1]¹ maps to the same pixel regardless of λ. You recover the direction only, not the distance.
  • 1
    One camera: a ray from the optical centre through pixel p. Every point on that ray gives the same image pixel. Depth is lost.
  • 2
    Two cameras: each produces its own ray. The unique intersection of those rays is the 3D point X — this is triangulation.
X₁ (near) and X₂ (far) both project to identical pixel p — depth is lost
Two cameras → two rays → unique 3D intersection
Two rays from O₁ and O₂ intersect at unique 3D point X
  • 1
    Camera 1 observes p → ray R₁ from O₁ through p
  • 2
    Camera 2 observes q → ray R₂ from O₂ through q
  • 3
    X = R₁ ∩ R₂ — the unique 3D location
This requires knowing R and t between the two cameras. That relationship is encoded in the Essential Matrix E.
p–q Relationship — Derivation (Lecture slides p.5–13)
Step 1 — Express x¹ in Camera 1 using unknown depth λ

If Camera 1 observes pixel p = (p₁, p₂), the 3D point lies somewhere along a ray. We express it with unknown scale λ (the depth):

// p̃ = homogeneous form [p₁, p₂, 1]¹
// λ = unknown depth (scale factor)

x¹ = λ · p̃ = λ[p₁, p₂, 1]¹
Why λ? We know the direction from p, but not how far along that direction X lies. Any value of λ gives the same image pixel. Depth is the unknown we want to recover.
Why homogeneous? Appending a 1 converts the 2D point into 3D, allowing rotation and translation to be applied as a single matrix multiply.
All points with different λ on the same ray give the same pixel p
Step 2 — Transform to Camera 2 → μq = λRp̃ + t

The rigid-body transform from Camera 1 to Camera 2 (rotation R, translation t) gives the point in Camera 2's frame. Substituting x¹ = λp̃:

x² = Rx¹ + t
x² = λRp̃ + t

// Project to image (unknown scale μ appears):
μq = λRp̃ + t
What this says (slide p.10): q is a linear combination of Rp̃ and t. Both λ and μ are unknown, but this structure encodes that q must lie on a specific line.
What is Rp̃? It is the direction of p, rotated into Camera 2’s coordinate frame. It represents where Camera 2 “sees” the same ray direction.
What is t? t is the position of O₁ expressed in Camera 2’s frame. This is exactly the epipole e₂ in Camera 2’s image.
q is a linear combo of Rp̃ (direction rotated to Cam2) and t (baseline)
Step 3 — The epipolar line ℓ = t × Rp̃
t = e₂ (epipole in Cam 2). Rp̃ = point on epipolar line. ℓ = t×Rp̃
  • t
    t is the position of O₁ in Camera 2’s frame
    Projectively equivalent to e₂ — the epipole in Camera 2 (where Cam 1 appears from Cam 2).
  • Rp̃
    Rp̃ is the direction of p rotated into Cam 2’s frame
    Equivalent to a point on the epipolar line.
  • ℓ
    Epipolar line = line through two points = their cross product
    ℓ = t × Rp̃
    q must satisfy q̃ · ℓ = 0.
Essential Matrix E = [t]ₓR — From Geometry to One Equation
Cross product as a matrix (slide p.15)

Epipolar line ℓ = t×Rp̃. To factor out p̃ cleanly, we replace the cross product with a matrix multiplication using the skew-symmetric matrix [t]ₓ:

// Expand t × x component by component:
row 1: −t₃x₂ + t₂x₃
row 2:  t₃x₁ − t₁x₃
row 3: −t₂x₁ + t₁x₂

// Exactly equals this matrix × x:
t × x = [[0,−t₃,t₂],[t₃,0,−t₁],[−t₂,t₁,0]] · x
     = [t]ₓ · x
Why is this possible? Each component of t×x is linear in x (no x² or x·y terms). Every linear map on a vector can be written as a matrix. The entries of [t]ₓ are just the components of t rearranged with signs.
t×Rp̃ = [t]ₓ(Rp̃) = ([t]ₓR)p̃ → define E = [t]ₓR
Full derivation chain → epipolar constraint q̃¹Ep̃ = 0
① Epipolar line (slide p.13)
ℓ = t × Rp̃ (line through e₂=t and Rp̃)
② Replace × with matrix (slide p.15)
ℓ = [t]ₓ · Rp̃ (skew-symmetric matrix form)
③ Factor → define E (slide p.16)
ℓ = ([t]ₓR) · p̃ = E · p̃ where E = [t]ₓR
④ q lies on ℓ (slide p.17)
q̃ · ℓ = 0 → q̃¹(Ep̃) = 0 → q̃¹Ep̃ = 0
A point q̃ lies on line ℓ iff their dot product = 0 in homogeneous coordinates.
E = [t]ₓ · R     q̃¹ E p̃ = 0
Why does Ep̃ give a LINE, not a point? p corresponds to infinitely many 3D depths λ. Each λ maps to a different location in Camera 2 — but all those locations lie on the same epipolar line. So E·p̃ = ℓ is a 3-vector of line coefficients, not a pixel.
E vs F: E works with normalised coords (focal length = 1). For pixel coords, use the Fundamental Matrix: F = K₂¹&supT; E K₁¹ which folds in camera calibration matrices K₁, K₂.
E·p̃ → line ℓ in Cam 2. q lies on ℓ ⇔ q̃¹Ep̃ = 0
Why E·ẽ₁ = 0? — The epipole and E’s null space (slide p.19)
All epipolar lines converge at e₁. E·ẽ₁ = 0 because Rẽ₁ ∥ t
when p = e₁
Ray through e₁ points exactly along the baseline (t direction)
therefore
Rẽ₁ is parallel to t → Rẽ₁ = αt
cross product of parallel vectors
t × αt = 0
result
Eẽ₁ = [t]ₓRẽ₁ = t × Rẽ₁ = 0
Practical implication: SVD(E) → the right singular vector for the smallest singular value = e₁. This also shows rank(E) = 2 (one singular value is zero).
8-Point Algorithm — Computing F from Point Correspondences
Degrees of Freedom: why exactly 8 points?
  • 1
    F is 3×3 → 9 entries
  • 2
    Scale invariance: −1 DOF
    F and kF encode identical epipolar geometry. Fix ‖F‖ = 1 → 8 free parameters.
  • 3
    Rank-2 constraint: −1 more DOF (nonlinear)
    det(F) = 0 because the epipole is in F’s null space. This is nonlinear — handled after solving by zeroing σ₃ in SVD(F).
  • →
    8 linear DOF → need 8 equations → 8 point pairs
    Each pair gives exactly 1 scalar equation: q̃¹Fp̃ = 0.
Why 1 equation per point? q̃¹Fp̃ = 0 is a scalar — one number. Compare homography: q = Hp gives 2 equations (x and y). Weaker constraint → need more points.
9 − 1(scale) = 8 linear DOF → 8 points needed. Rank-2 enforced afterwards.
Expanding q̃¹Fp̃ = 0 into one linear row
// Full expansion of q̃¹ F p̃ = 0 (one correspondence → one equation):
q₁p₁·F₁₁ + q₁p₂·F₁₂ + q₁·F₁₃ +
q₂p₁·F₂₁ + q₂p₂·F₂₂ + q₂·F₂₃ +
p₁·F₃₁ + p₂·F₃₂ + F₃₃ = 0

// Reshape as dot product with unknown vector f = [F₁₁…F₃₃]¹:
[q₁p₁, q₁p₂, q₁, q₂p₁, q₂p₂, q₂, p₁, p₂, 1] · f = 0

Stack 8 such rows (from 8 point pairs) into matrix A (8×9). Solve Af = 0 by SVD: last column of V. Reshape into 3×3, then enforce rank-2 by zeroing σ₃ in a second SVD.

Same structure as Week 4! The homography also solved Ah = 0 by SVD. Difference: H uses 4 points×2 equations = 8 rows. F uses 8 points×1 equation = 8 rows. Identical algebra, different constraint per point.
Stereo Reconstruction — Parallel Cameras & Disparity
E for parallel cameras → epipolar lines are horizontal rows (L13 slides p.5–8)

For a pure x-translation stereo pair (R = I, t = [a, 0, 0]¹):

E = [t]ₓR = [t]ₓ = [[0,0,0],[0,0,−a],[0,a,0]]

// For point p = [u₁, v₁, 1]¹:
ℓ = Ep̃ = [0, −1, v₁]¹

// Line equation: 0x − y + v₁ = 0 → y = v₁
Key result: q must lie on the same horizontal row as p. 2D search → 1D search — a massive computational win for stereo matching.
R=I, t=[a,0,0] → ℓ=[0,−1,v₁]¹ → y=v₁ for every epipolar line
Deriving d = b·f / Z via similar triangles (L13 slides p.13–16)
  • 1
    Camera 1 projection (similar triangles, normalised f=1):
    u₁/1 = a₁/z → u₁ = a₁/z
  • 2
    Camera 2 projection (u₂ is negative — left of principal point):
    u₂/1 = a₂/z → u₂ = a₂/z
  • 3
    Subtract → X_w cancels:
    d = u₁ − u₂ = (a₁+a₂)/z = a/z
    Both u₁ and u₂ contain the world-x position, which cancels on subtraction.
  • 4
    With focal length f pixels:
    d = a·f / z
disparity  d = b · f / Z
Similar triangles → d = a/z → depth z = a/d

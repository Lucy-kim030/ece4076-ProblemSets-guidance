Background

What is Unsupervised Learning?

In supervised learning, every data point has a label — "this pixel is a road", "this image contains a cat". Unsupervised learning asks a harder question: what if we have no labels at all? Can we still discover meaningful structure in the data?

🎯
ClusteringGroup similar data points together, without knowing the group meanings in advance
📐
Distance FunctionsThe choice of distance (ℓ₁ vs ℓ₂) changes what "similar" means and who the optimal centre is
📊
Density EstimationModel the data's probability distribution — Gaussian is the most common choice
🔍
Vision ApplicationsImage segmentation, background modelling, feature clustering in visual recognition
💡 Core idea of this problem set: K-Means finds cluster centres by minimising the total squared Euclidean distance to all assigned points. This problem set proves why the arithmetic mean is the unique optimal centre — and explores what changes when we switch to ℓ₁ distance.
Question 1

Gradient of the Norm — Why K-Means Uses the Mean

This question builds the calculus foundation of K-Means from scratch. Four sub-parts chain together: first we express norms as dot products, then compute gradients, then use those gradients to prove that the optimal cluster centre is the arithmetic mean.

Part (a) — Show that ‖µ‖² = µᵀµ

This establishes that the squared L2 norm equals the vector dot product with itself — a fact that makes differentiating norms much simpler in the parts that follow.

Part (a)
Expand µᵀµ
Write out product
µᵀµ = µ₁·µ₁ + µ₂·µ₂ + ··· + µₙ·µₙ
Simplify squares
= µ₁² + µ₂² + ··· + µₙ²
Recognise definition
= ‖µ‖² ✓matches the definition of squared L2 norm
🎓 Why this matters: Writing norms as dot products converts geometry into linear algebra, which has clean differentiation rules we can apply in parts (b) and (c).

Part (b) — Show that ∂‖µ‖²/∂µ = 2µ

The gradient of a scalar function over a vector is itself a vector — each component is the partial derivative with respect to one element of µ. This is the multivariate analogue of the scalar rule d(x²)/dx = 2x.

Part (b)
Differentiate component j
Expand from (a)
‖µ‖² = µ₁² + ··· + µⱼ² + ··· + µₙ²
∂/∂µⱼ each term
= 0 + ··· + 2µⱼ + ··· + 0only µⱼ² depends on µⱼ
Stack into gradient vector
All n components
∂‖µ‖²/∂µ = [ 2µ₁, 2µ₂, …, 2µₙ ]ᵀ
Factor out 2
= 2µ ✓same as scalar rule d(x²)/dx = 2x, applied per component

Part (c) — Show that ∂‖x−µ‖²/∂µ = −2(x−µ)

This is the critical gradient for K-Means: how does the squared distance between a fixed point x and the centre µ change as we move µ? The negative sign tells us which direction to shift µ to reduce the distance.

Part (c)
Step 1 — expand the norm
Use (a−b)ᵀ(a−b)
‖x−µ‖² = xᵀx − 2µᵀx + µᵀµ
Identify terms
= ‖x‖² − 2µᵀx + ‖µ‖²
Step 2 — differentiate each term w.r.t. µ
∂/∂µ of ‖x‖²
= 0x is a fixed constant, does not depend on µ
∂/∂µ of 2µᵀx
= 2xµᵀx = Σⱼ µⱼxⱼ, so ∂/∂µⱼ = xⱼ → stack into 2x
∂/∂µ of ‖µ‖²
= 2µfrom part (b)
Step 3 — combine
Sum the parts
∂‖x−µ‖²/∂µ = 0 − 2x + 2µ = 2µ − 2x
Factor
= −2(x−µ) ✓
⚠️ Mind the sign: The gradient −2(x−µ) is negative when µ < x, meaning it points leftward — away from x. Moving µ in the negative gradient direction therefore pulls µ toward x, which is exactly what we want a clustering algorithm to do.

Part (d) — Prove µ* = (1/m) Σᵢ xᵢ (the arithmetic mean)

Using the gradient from part (c), we now prove that the unique minimiser of the total squared distance is the arithmetic mean — the mathematical justification for K-Means' centroid update step.

Part (d)
Differentiate the total loss
Total loss
∂L/∂µ = ∂/∂µ Σᵢ ‖xᵢ−µ‖²
Linearity of ∂
= Σᵢ ∂/∂µ ‖xᵢ−µ‖²derivative of sum = sum of derivatives
Apply part (c)
= Σᵢ −2(xᵢ−µ)
Set gradient = 0 and solve
Set to zero
Σᵢ −2(xᵢ−µ*) = 0
Expand sum
−2 Σᵢ xᵢ + 2 Σᵢ µ* = 0
µ* is constant over i
Σᵢ xᵢ = m · µ*Σᵢ µ* = m·µ* since µ* doesn't change with i
Divide by m
µ* = (1/m) Σᵢ xᵢ ✓the arithmetic mean!
µ* = (x₁ + x₂ + ⋯ + xₘ) / m — the mean is the unique optimal centre under ℓ₂² loss
🎓 Why this matters for K-Means: The "update centroid to the average" step isn't a heuristic — it's the exact closed-form minimiser. Each iteration is therefore guaranteed to reduce (or maintain) the total loss.

Question 2

ℓ₁ Distance and K-Medians

What changes when we replace squared Euclidean distance with the Manhattan (ℓ₁) distance? Everything — the geometry of the unit ball, the structure of the derivative, and the identity of the optimal centre all shift fundamentally.

Part (a) — Plot the ℓ₁ unit ball in ℝ²

In ℓ₂, all points at distance 1 from the origin form a circle. Under ℓ₁, the same set forms a rotated square (diamond), because the constraint |x₁| + |x₂| = 1 is piecewise-linear — it defines a straight line in each of the four quadrants.

Part (a)
Analyse each quadrant — remove |·| using the sign in that region
Quadrant I (x₁≥0, x₂≥0)
x₁ + x₂ = 1 → line from (1,0) to (0,1), slope = −1
Quadrant II (x₁<0, x₂≥0)
−x₁ + x₂ = 1 → line from (−1,0) to (0,1), slope = +1
Quadrant III (x₁<0, x₂<0)
−x₁ − x₂ = 1 → line from (−1,0) to (0,−1), slope = −1
Quadrant IV (x₁≥0, x₂<0)
x₁ − x₂ = 1 → line from (1,0) to (0,−1), slope = +1
Result
Four line segments → diamond with vertices at (±1,0) and (0,±1) ✓

Part (b) — Compute d/dµ |x − µ|

Unlike the smooth quadratic in Q1, the absolute value function has a sharp kink at x = µ. We handle the two smooth regions separately, then note what happens at the kink.

Part (b)
Case 1: x > µ, so (x−µ) > 0
Remove |·|
|x−µ| = x−µ
Differentiate w.r.t. µ
d/dµ (x−µ) = −1
Case 2: x < µ, so (x−µ) < 0
Remove |·|
|x−µ| = −(x−µ) = µ−x
Differentiate w.r.t. µ
d/dµ (µ−x) = +1
Summary
Piecewise result
d/dµ |x−µ| = −1 if x>µ, +1 if x<µ, undefined at x=µ
⚠️ At x = µ: the function has a sharp corner — the classical derivative does not exist. In optimisation theory we use a subgradient (any value in [−1,+1] is valid here). The minimum sits exactly at this kink, so we don't need the precise value there.

Part (c) — Prove that µ* is the median

By switching from ℓ₂ to ℓ₁ distance, the optimal centre shifts from the mean to the median. The key is that every data point now contributes a vote of exactly ±1, regardless of distance.

Part (c)
Differentiate total ℓ₁ loss
Total loss
L(µ) = Σᵢ |xᵢ−µ|
Apply part (b)
dL/dµ = Σᵢ { −1 if xᵢ>µ, +1 if xᵢ<µ }
Let A = #(xᵢ above µ), B = #(xᵢ below µ)
dL/dµ = −A + B
Set to zero and interpret
Set gradient = 0
−A + B = 0 ⟹ A = B
Interpretation
µ* = median ✓equal number of points above and below µ* — that's the definition of median
µ* = median — exactly half the data lies above, half below
🎓 The key contrast with Q1: In ℓ₂, the derivative −2(xᵢ−µ) grows linearly with distance, so a far outlier exerts a massive pull. In ℓ₁, every point contributes exactly ±1, so an outlier at distance 1000 has precisely the same pull as a neighbour at distance 1.

Part (d) — When to prefer K-Medians over K-Means?

Part (d)

K-Medians is preferred when the data is expected to contain outliers. In K-Means, a single point far from the cluster can dramatically pull the centroid away from the true cluster centre (squared distance amplifies the contribution of distant points). In K-Medians, every point contributes exactly ±1 regardless of distance, so extreme values cannot distort the result.

✅ Use K-Medians when: data has sensor noise, mislabelled samples, or spatial outliers (common in computer vision). Use K-Means when: data is clean and you want the simpler algorithm — mean is O(n), median is O(n log n) per iteration.

Question 3

The K-Means Decision Boundary

Given cluster centres µ₁ = (−1, 2)ᵀ and µ₂ = (2, 3)ᵀ, any new point in ℝ² is assigned to whichever centre is closer. The decision boundary is the set of all points exactly equidistant from both — geometrically the perpendicular bisector of the segment joining µ₁ and µ₂.

Full Solution
Set d²(p,µ₁) = d²(p,µ₂) with µ₁=(−1,2), µ₂=(2,3)
Expand left side
(x+1)² + (y−2)² = x²+2x+1 + y²−4y+4
Expand right side
(x−2)² + (y−3)² = x²−4x+4 + y²−6y+9
Cancel x² and y² from both sides, collect terms
After cancelling
2x+1−4y+4 = −4x+4−6y+9
Move all to left
2x+4x − 4y+6y + 1+4−4−9 = 0
Simplify
6x + 2y − 8 = 0
Divide by 2
3x + y − 4 = 0 ✓equivalently y = −3x + 4 (slope −3, y-intercept 4)
Decision boundary: 3x + y − 4 = 0
📐 Sanity check with midpoint: The midpoint of µ₁ and µ₂ is (0.5, 2.5). Substituting: 3(0.5) + 2.5 − 4 = 0 ✓ The midpoint always sits exactly on the boundary, confirming the result.

Question 4

Computing the Mean Vector and Covariance Matrix

Given four data points x₁=(10,9)ᵀ, x₂=(10,11)ᵀ, x₃=(8,9)ᵀ, x₄=(12,11)ᵀ, we fit a 2D Gaussian by computing its two parameters: the mean vector µ (where the distribution is centred) and the covariance matrix Σ (how the data spreads and how the two dimensions are correlated).

Full Solution
Step 1 — Mean vector µ = (1/4) Σ xᵢ
x-component
µ_x = (10+10+8+12)/4 = 40/4 = 10
y-component
µ_y = (9+11+9+11)/4 = 40/4 = 10
Mean vector
µ = (10, 10)ᵀ
Step 2 — Variance of x: σ²_x = (1/4) Σ (xᵢ−µ_x)²
Squared deviations
(10−10)²+(10−10)²+(8−10)²+(12−10)² = 0+0+4+4 = 8
Divide by 4
σ²_x = 8/4 = 2
Step 3 — Variance of y: σ²_y = (1/4) Σ (yᵢ−µ_y)²
Squared deviations
(9−10)²+(11−10)²+(9−10)²+(11−10)² = 1+1+1+1 = 4
Divide by 4
σ²_y = 4/4 = 1
Step 4 — Covariance: σ_xy = (1/4) Σ (xᵢ−µ_x)(yᵢ−µ_y)
Products of deviations
(0)(−1) + (0)(+1) + (−2)(−1) + (2)(+1) = 0+0+2+2 = 4
Divide by 4
σ_xy = 4/4 = 1
Step 5 — Assemble Σ
Covariance matrix
Σ = [ 2 1 ]
          [ 1 1 ]
🔑 Interpreting σ_xy = +1: Positive covariance means x and y increase together. When x is above average (x=12), y is also above average (y=11); when x is below (x=8), y is also below (y=9). This positive co-movement tilts the fitted Gaussian's ellipse toward the upper-right diagonal.

Question 5

Writing Down the Multivariate Gaussian PDF

With µ and Σ computed, we can write the full probability density function. The PDF tells us the relative likelihood of observing any point (x,y) — higher density means the point is more "typical" under the fitted distribution.

Full Solution
Step 1 — Compute det(Σ)
For Σ=[2,1;1,1]
|Σ| = (2)(1) − (1)(1) = 2 − 1 = 1
Step 2 — Compute Σ⁻¹ = (1/det) · [d,−b;−c,a]
Swap diagonal, negate off-diag, ÷ by det=1
Σ⁻¹ = [ 1 −1 ]
           [ −1 2 ]
Step 3 — Normalisation constant (N=2, |Σ|=1)
Plug in N=2, |Σ|=1
1/((2π)^(2/2) · 1^(1/2)) = 1/(2π)
Step 4 — Write the full PDF
Final result
P(x,y) = (1/2π) · exp(−½ · [x−10, y−10] · [1,−1;−1,2] · [x−10, y−10]ᵀ)
P(x,y) = (1/2π) · exp(−½ · [x−10, y−10] · [1,−1;−1,2] · [x−10, y−10]ᵀ)
💡 The term (x−µ)ᵀ Σ⁻¹ (x−µ) inside the exponential is the Mahalanobis distance squared — the "normalised" distance from point (x,y) to the mean that accounts for the shape and orientation of the distribution. Points at the same Mahalanobis distance form an ellipse, which are the probability contour lines.

Question 6

Evaluating the PDF at the Four Original Data Points

We now substitute each of the four data points into the PDF derived in Q5. A remarkable symmetry emerges: because all four points are symmetric around the mean (10,10), they all have the same Mahalanobis distance, and therefore the same probability density.

Full Solution
Evaluate P(10,9) — compute deviation, Mahalanobis², then PDF
Step 1 — Deviation from mean
v − µ = (10−10, 9−10)ᵀ = (0, −1)ᵀ
Step 2 — Apply Σ⁻¹
Σ⁻¹(v−µ) = [1·0+(−1)(−1), (−1)·0+2·(−1)]ᵀ = (1, −2)ᵀ
Step 3 — Mahalanobis²
[0,−1]·(1,−2)ᵀ = (0)(1)+(−1)(−2) = 2
Step 4 — Exponential
exp(−½·2) = e⁻¹ ≈ 0.3679
Step 5 — Full PDF
P(10,9) = (1/2π)·e⁻¹ ≈ 0.0585

All four points have the same Mahalanobis distance — here's why:

Verify Mahal²=2 for all four points (deviations from µ)
x₁=(10,9): v−µ=(0,−1)
[0,−1]·Σ⁻¹·[0,−1]ᵀ = 2 ✓
x₂=(10,11): v−µ=(0,+1)
[0,+1]·Σ⁻¹·[0,+1]ᵀ = [0,1]·(−1,2)ᵀ = 2 ✓
x₃=(8,9): v−µ=(−2,−1)
[−2,−1]·Σ⁻¹·[−2,−1]ᵀ = [−2,−1]·(−1,0)ᵀ = 2 ✓
x₄=(12,11): v−µ=(+2,+1)
[+2,+1]·Σ⁻¹·[+2,+1]ᵀ = [+2,+1]·(1,0)ᵀ = 2 ✓
Therefore
P = (1/2π)·e⁻¹ ≈ 0.0585 for all four points ✓
P(10,9) = P(10,11) = P(8,9) = P(12,11) = (1/2π)·e⁻¹ ≈ 0.0585
🎓 Why they're all equal: All four points lie on the same probability contour ellipse of the fitted Gaussian — they are all equally "typical" under the model, because the model was fitted to those exact four points symmetrically placed around the mean.

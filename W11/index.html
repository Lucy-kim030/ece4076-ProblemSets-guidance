01
Intersection over Union (IOU)
What is IOU?

IOU measures how well a predicted box overlaps a ground-truth box. It is the ratio of their intersection (overlap area) to their union (total covered area).

IOU = area(Bp ∩ Bgt) / area(Bp ∪ Bgt)
// union = S1 + S2 - intersection (avoids double-counting)

Intersection width = min(x2, a2) - max(x1, a1)
Intersection height = min(y2, b2) - max(y1, b1)
// if either dimension ≤ 0 → no overlap → IOU = 0
IOU = 1.0 → perfect overlap
IOU ≥ 0.5 → accepted as correct
IOU < 0.5 → incorrect detection

Worked Example — Problem Set Q1(2)

Given: x₁=b₁=1, y₁=a₁=2, x₂=b₂=3, y₂=a₂=4

①
Box areas
S₁ (GT) = (x₂−x₁)×(y₂−y₁) = (3−1)×(4−2) = 2×2 = 4
S₂ (Pred) = (a₂−a₁)×(b₂−b₁) = (4−2)×(3−1) = 2×2 = 4
②
Intersection box corners
Left boundary = max(x₁,a₁) = max(1,2) = 2
Right boundary = min(x₂,a₂) = min(3,4) = 3 → width = 1
Top boundary = max(y₁,b₁) = max(2,1) = 2
Bottom boundary = min(y₂,b₂) = min(4,3) = 3 → height = 1
S_x = 1×1 = 1
③
IOU
IOU = S_x / (S₁ + S₂ − S_x) = 1 / (4+4−1) = 1/7 ≈ 0.14
→ IOU < 0.5 → Incorrect detection ✗
02
Non-Maximum Suppression (NMS)
Why NMS?

Detectors generate thousands of overlapping candidate boxes per image. Without cleanup, a single dog might have 50 red boxes around it. NMS keeps only the most confident box per object.

NMS Algorithm Steps
1
Collect all predictions
Gather every bounding box for a specific class, each with a confidence score. (Run separately per class.)
2
Sort by confidence ↓
Order boxes from highest to lowest confidence score. The most-certain box goes first.
3
Pick the top box
Select the box with the highest remaining confidence and add it to the final output list.
4
Suppress overlapping boxes
Remove all remaining boxes whose IOU with the selected box exceeds the threshold (typically 0.5 or 0.7).
5
Repeat until empty
Go back to Step 3 with remaining boxes. Continue until no boxes are left.

Precision, Recall & mAP
Key Concepts
TRUE POSITIVE (TP)
Predicted "object here" AND IOU ≥ threshold. Correct hit.
FALSE POSITIVE (FP)
Predicted "object here" BUT IOU < threshold. Incorrect detection.
FALSE NEGATIVE (FN)
A real object was not detected (missed). Increases recall penalty.
GROUND TRUTH (GT)
Total real objects in the dataset. Denominator for Recall.
Precision = TP / (TP + FP) // of what I found, how many are correct?
Recall = TP / (TP + FN) // of all real objects, how many did I find?

AP Calculation
Raw PR curve Interpolated curve (step) 11 sample points

The 11-point interpolation samples recall at {0.0, 0.1, ..., 1.0} and at each point takes the maximum precision seen at that recall or higher.

Precision_interp(r) = max_{r̂ ≥ r} Precision(r̂)

AP = (1/11) × Σ Precision_interp(r) for r ∈ {0, 0.1, ..., 1.0}

From Fig. 6 (Pascal VOC2008 example):

①
Read off interpolated precision at each of 11 points:
r = 0.0→0.4: P_interp = 1.0 (5 points)
r = 0.5→0.8: P_interp = 0.57 (4 points)
r = 0.9→1.0: P_interp = 0.50 (2 points)
②
Average:
AP = (5×1.0 + 4×0.57 + 2×0.50) / 11
= (5.0 + 2.28 + 1.00) / 11
= 8.28 / 11 ≈ 0.75
③
mAP:
Repeat AP computation for every object class, then take the mean.
mAP = average(AP_car, AP_person, AP_dog, ...)
Final Answer
AP = (5×1.0 + 4×0.57 + 2×0.50) / 11
AP ≈ 0.75
04
Summary
IOU
Overlap ratio between predicted & ground-truth box. Threshold (usually 0.5) decides TP vs FP.
intersection / union
NMS
Removes duplicate boxes. Sort by confidence → pick top → suppress overlapping → repeat.
per-class greedy filter
Precision
Of all my detections, how many are correct? High precision = few false alarms.
TP / (TP + FP)
Recall
Of all real objects, how many did I find? High recall = few missed objects.
TP / (TP + FN)
AP
Area under the PR curve for one class. 11-point interpolation smooths jagged curves.
avg of 11-point P_interp
mAP
Mean AP across all object classes. The standard benchmark metric for detectors.
mean(AP_class1, AP_class2, ...)
